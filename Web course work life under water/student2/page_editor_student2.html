2. Design & Implementation Rationale

Colour Palette: Deep navy (#020d1a) and vibrant cyan (#00d4ff) were selected to evoke the ocean environment and reinforce the SDG 14 theme. The palette provides strong contrast ratios (> 4.5:1), meeting WCAG AA accessibility standards.

Typography: Google Fonts' Poppins was chosen for its clean geometric style and wide weight range, allowing clear visual hierarchy through weight variation rather than multiple font families.

splash.html – Technical Choices:

  • The <meta http-equiv="refresh"> tag with content="4;url=home.html" implements the required 4-second auto-redirect without relying on JavaScript, ensuring it works even if JS is disabled.
  • The CSS loader uses four <div> children inside .loader-ring, styled with @keyframes ring-spin and staggered animation-delay values—purely CSS, no images or GIFs.
  • The JavaScript countdown uses setInterval with a 1000ms interval, decrementing a remaining variable and updating the DOM. clearInterval is called before any redirect to prevent ghost ticks.

ais.html – Technical Choices:

  • Cards use role="checkbox" and aria-checked attributes for screen-reader accessibility. tabindex="0" ensures keyboard navigation.
  • Dynamic backgrounds are applied via body.style.backgroundImage and a corresponding overlay colour shift, giving three distinct visual states (polluted / recovering / thriving).
  • CSS Grid (auto-fill, minmax(300px, 1fr)) provides a responsive card layout that adapts from 3 columns on desktop to 1 column on mobile without media queries for the grid itself.
  • The score progress bar uses role="progressbar" with aria-valuenow updated in real time, providing screen-reader feedback on score changes.

3. Reflection – Challenges & Lessons Learned

The most challenging aspect of the splash screen was ensuring the countdown timer and the meta-refresh tag remained in sync. The meta refresh fires independently of JavaScript, so a race condition could cause the page to redirect before the countdown visually reached zero. This was resolved by setting the setInterval tick interval to exactly 1000 ms, matching the meta refresh seconds value, and calling clearInterval before any programmatic redirect.

⚠ Key Challenge: Getting the CSS-only animation loader to remain visible without disrupting the page layout on smaller viewports. Resolved by using position: fixed for the loader, decoupling it from document flow.

Building the AIS helped deepen understanding of JavaScript event delegation, ARIA accessibility attributes, and how CSS transitions can be driven dynamically via style manipulation in JS. The dynamic background feature demonstrated how a simple backgroundImage style assignment combined with a CSS transition property creates a smooth, professional visual experience without any library.
