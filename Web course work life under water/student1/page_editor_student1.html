2. Design & Implementation Rationale

Shared Template: The navigation header and footer are implemented consistently across all pages using shared CSS design tokens via CSS custom properties (variables), ensuring a unified visual language throughout the site.

home.html: Uses semantic HTML5 elements including <header>, <main>, <section>, <article>, and <footer>. A sticky header with backdrop-filter:blur() provides a glassmorphism effect without JS. The stats strip uses CSS Grid for responsive layout. The IntersectionObserver API drives scroll-reveal animations for cards and a counter animation for statistics.

gallery.html: The filter system uses data-cat attributes on <article> elements, toggled by JavaScript without any framework. The lightbox is a custom modal using position:fixed with keyboard navigation (Escape, ArrowLeft, ArrowRight) and focus management for accessibility. Images use loading="lazy" for performance.

3. Reflection

The most challenging aspect was achieving consistent cross-browser support for backdrop-filter on the sticky nav bar across Chrome, Edge, and Firefox. The solution was to include both -webkit-backdrop-filter and backdrop-filter properties. Building the lightbox from scratch deepened understanding of focus trapping and keyboard event management for accessibility compliance.
