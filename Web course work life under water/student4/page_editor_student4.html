2. Design & Implementation Rationale

profile.html: The profile form persists all data using localStorage.setItem() and localStorage.getItem() with JSON serialisation. This allows the profile to survive page reloads without any server-side code. The avatar picker uses emoji characters as visual elements, eliminating image asset dependencies while remaining fully accessible via aria-label attributes. CSS toggle switches are implemented using a hidden checkbox, a styled span track, and a thumb that translates 20px on :checked state using CSS transitions.

sitemap.html: The SVG sitemap is hand-authored using SVG primitives: <rect>, <line>, <text>, and <linearGradient> elements. <marker> with a path is used for directional arrow heads. The <a> SVG element wraps each node to make them navigable. The SVG is given role="img" with a descriptive aria-label, and individual groups have role="link" with aria-label for screen readers.

3. Reflection

The most challenging aspect was designing the SVG sitemap so it rendered clearly at all viewport sizes whilst remaining clickable. Using viewBox with a fluid width:100% SVG and min-width:700px on the container ensures horizontal scrolling on small screens rather than illegible diagram compression. The profile localStorage implementation required careful handling of the "first visit" state where no stored data exists.
