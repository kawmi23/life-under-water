2. Design & Implementation Rationale

feedback.html: Client-side form validation uses the novalidate attribute to disable browser defaults, allowing custom error messages styled consistently with the site theme. Each field validation is triggered on blur and on submit. The star rating uses CSS-only flex-direction:row-reverse with :checked~label selectors to achieve the interactive star highlight effect without JavaScript. The character counter uses an input event listener.

team.html: Team cards use CSS Grid with auto-fill and minmax for responsive layout. The IntersectionObserver API drives an entrance animation for each card as it scrolls into view, improving perceived performance by avoiding a large upfront render.

localStorage Stats: Feedback submission counts and ratings are persisted in localStorage so community stats survive page refreshes without a backend server.

3. Reflection

The star rating implementation using pure CSS (flex-direction:row-reverse with sibling selectors) was the most technically innovative part. It required careful understanding of how CSS sibling combinators work in reverse order. Accessibility was a key challenge – ARIA live regions for error messages and role="alert" ensure screen readers announce validation errors without requiring page reload.
